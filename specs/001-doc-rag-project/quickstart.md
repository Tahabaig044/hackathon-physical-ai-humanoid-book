# Quickstart Guide: Physical AI & Humanoid Robotics Textbook Project

This guide provides a quick overview to get started with the Physical AI & Humanoid Robotics Textbook project, focusing on setting up the Docusaurus frontend and the FastAPI RAG backend.

## 1. Project Structure

The project is conceptually divided into two main parts:

-   **Docusaurus Frontend**: The static site generated by Docusaurus, serving the textbook content.
-   **FastAPI RAG Backend**: The API service for the Retrieval-Augmented Generation (RAG) chatbot, handling document retrieval and conversational AI.

## 2. Setting up the Docusaurus Frontend

### Prerequisites

-   Node.js (latest LTS)
-   npm or Yarn

### Steps

1.  **Navigate to the Docusaurus project directory (once created, e.g., `physical-ai-book`)**:
    ```bash
    cd physical-ai-book
    ```
2.  **Install dependencies**:
    ```bash
    npm install # or yarn
    ```
3.  **Start the development server**:
    ```bash
    npm start # or yarn start
    ```
    The Docusaurus site will be available at `http://localhost:3000`.

4.  **Build the static site**:
    ```bash
    npm run build # or yarn build
    ```
    The static assets will be generated in the `build/` directory.

## 3. Setting up the FastAPI RAG Backend

### Prerequisites

-   Python 3.11+
-   Poetry (recommended for dependency management) or pip
-   Neon Postgres database instance (connection string required)
-   Qdrant vector database instance (connection details required)
-   OpenAI API Key (or compatible LLM API key)

### Steps

1.  **Navigate to the FastAPI project directory (once created, e.g., `rag-backend`)**:
    ```bash
    cd rag-backend
    ```
2.  **Set up virtual environment and install dependencies (using Poetry)**:
    ```bash
    poetry install
    poetry shell
    ```
    Or using pip:
    ```bash
    python -m venv venv
    ./venv/Scripts/activate # On Windows
    source venv/bin/activate # On macOS/Linux
    pip install -r requirements.txt
    ```
3.  **Configure environment variables**: Create a `.env` file in the `rag-backend` directory with the following (example values):
    ```ini
    DATABASE_URL="postgresql://user:password@host:port/database"
    QDRANT_HOST="localhost"
    QDRANT_PORT="6333"
    OPENAI_API_KEY="sk-..."
    ```
4.  **Run database migrations (if applicable)**:
    ```bash
    # Example using Alembic or similar
    alembic upgrade head
    ```
5.  **Start the FastAPI application**:
    ```bash
    uvicorn main:app --reload
    ```
    The FastAPI application will be available at `http://localhost:8000`. The API documentation (Swagger UI) can be accessed at `http://localhost:8000/docs`.

## 4. Integrating the Docusaurus Frontend with the RAG Backend

Once both the frontend and backend are running, the Docusaurus site can make API calls to the FastAPI backend to utilize the RAG chatbot functionality. This will typically involve:

-   Implementing a chat interface within Docusaurus.
-   Making authenticated requests from the Docusaurus frontend to the `/api/v1/chat/conversations/{conversation_id}/messages` endpoint.
-   Displaying chatbot responses and potentially linking to retrieved documents.
